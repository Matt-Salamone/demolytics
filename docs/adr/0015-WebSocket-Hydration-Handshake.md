# **0015: WebSocket Hydration Handshake**

**Context:** The Presentation Layer (Windows Game Bar Widget) is ephemeral; it can be mounted, unmounted, or manually closed by the user at any point during a match. If the Engine only broadcasts state updates when discrete events occur (e.g., a goal is scored), a re-opened Widget will remain in an empty, desynchronized state until the next event triggers. Alternatively, continuously pushing the entire state payload at high frequencies wastes bandwidth and CPU.  
**Decision:** We will implement a strict "Hydration Handshake" over the local WebSocket. The Engine is designated as the sole Source of Truth. Upon connection or reconnection, the Presentation Layer will immediately emit a RequestSync event. The Engine will intercept this event and respond with a comprehensive StateHydration payload containing the current Live Match State, Live Session State, and any currently active Insights.  
**Consequences:** The UI will instantly snap to the correct state regardless of when it is opened, ensuring a seamless user experience. Network traffic is kept minimal, as heavy state payloads are only transmitted upon explicit UI request, leaving the WebSocket free for lightweight event triggers during normal gameplay.
