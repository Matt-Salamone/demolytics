# **0011: Categorical Cooldown Matrix for Insight Rendering**

**Context:** Utilizing pure Z-score math to rank Insights can result in "Boring Stat Dominance" (e.g., high-variance counting stats like Saves constantly overshadowing continuous rate stats like Average Speed). Conversely, implementing subjective "engagement multipliers" based on user feedback corrupts the strict mathematical objectivity of the Engine.  
**Decision:** We will implement a Categorical Cooldown Matrix. All statistics are bucketed into three canonical categories: Impact & Playmaking, Resource Management, and Pace & Positioning. The Engine will continue to rely strictly on Z-score normalization to find the highest mathematical outlier. However, once a stat "wins" and is displayed on the Podium UI, its parent category enters a cooldown state for the next configurable number of goal replays.  
**Consequences:** The Engine's math remains strictly objective and locally calculated. The UI is structurally forced to rotate through different aspects of the player's game (e.g., forcing a mechanics-based insight to display if a playmaking-based insight was just shown), preventing alert fatigue without needing subjective tuning algorithms.
